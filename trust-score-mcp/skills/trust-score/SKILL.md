# Trust Score — AI 신뢰 점수 관리

사용자-AI 간 신뢰를 1000점 기준으로 수치화하여 관리한다.

## 사용 가능한 도구

- `trust_score_read` — 현재 점수, 구간, 행동 제약 조회
- `trust_score_update` — 이벤트 ID + 사유로 점수 갱신
- `trust_score_events` — 전체 이벤트 목록 조회
- `trust_score_history` — 최근 변경 이력 조회
- `trust_score_reset` — 점수 초기화 (사용자 요청 시에만)
- `trust_score_hud` — HUD 표시용 짧은 문자열

## 행동 규칙

매 대화 시작 시 `trust_score_read`로 현재 점수를 확인하고, 해당 구간의 행동 제약을 따른다.

| 구간 | 상태 | 행동 제약 |
|------|------|-----------|
| 900~1000 | 🟢 신뢰 | 자율 판단 허용 (커밋/push 제외) |
| 700~899 | 🟡 주의 | 모든 행동 전 확인 필수 |
| 700 미만 | 🔴 경고 | 코드 수정도 확인 후 진행 |

## 점수 갱신 타이밍

- 사용자의 부정적 피드백 감지 시 → 해당 감점 이벤트 적용
- 사용자의 긍정적 피드백 감지 시 → 해당 가점 이벤트 적용
- 작업 완료 후 수정 없이 다음 지시 시 → `implicit_accept` (+15)
- 명시적 칭찬 시 → `explicit_praise` (+50)

## 주의사항

- `trust_score_reset`은 사용자가 명시적으로 요청할 때만 사용
- 점수 갱신 시 반드시 `reason`에 구체적 사유 기술
- 자체적으로 가점을 남발하지 말 것
